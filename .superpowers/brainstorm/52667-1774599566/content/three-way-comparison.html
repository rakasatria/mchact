What mchact-memory Should Take From Each

Best-of-three ingredients for the unified memory crate

From Honcho (observation model)

  • 4-level observation hierarchy — explicit/deductive/inductive/contradiction
  • Peer paradigm — unified user/agent identity with observer/observed
  • Deriver agent — async queue-based extraction, multi-level observations
  • Dreamer agent — offline consolidation, pattern discovery, contradiction detection
  • Source attribution DAG — source_ids linking premises to conclusions
  • Peer cards — max 40 durable biographical facts, auto-maintained
  • Hybrid RRF search — semantic + keyword merged via Reciprocal Rank Fusion
  • Background task queue — derive/dream/consolidate tasks processed async

From Hermes (production patterns)

  • Frozen memory snapshot — load at session start, stable system prompt, preserves prefix cache. Hermes proved this works for cost savings.
  • Memory security scanning — PII redaction, prompt injection detection, secrets guard before writes. mchact's quality gates are basic by comparison.
  • Session search + LLM summarization — FTS5 search is not enough; Hermes adds an auxiliary model to summarize matched sessions. More useful than raw snippets.
  • Curated memory with char caps — Hermes' MEMORY.md (2,200 chars) forces quality curation. Peer cards (40 facts) follow similar bounded principle.
  • MoA findings as promotable memory — Hermes doesn't do this yet, but mchact's findings blackboard + reflector can promote valuable findings to observations.
  • Token tracking per session — input/output/cache_read/cache_write/reasoning. mchact should track this for the memory crate's LLM calls (deriver/dreamer costs).

From mchact/microclaw (existing infrastructure)

  • FTS5 + sqlite-vec — already implemented, proven. SQLite driver reuses these.
  • 5-phase compressor — already designed. Memory crate doesn't replace this.
  • MoA findings blackboard — already designed. Moves into mchact-memory.
  • MemoryProvider trait — existing integration point. ObservationStore replaces it.
  • Reflector loop infrastructure — scheduler.rs background task pattern reused for deriver/dreamer.
  • Quality gates — memory_quality.rs normalization + rejection. Extended with Hermes-style security scanning.
  • Confidence scoring — 0.0-1.0 with source-based defaults. Preserved in observations.
