How Hybrid Search Works on Each Driver

SQLite: FTS5 + sqlite-vec + RRF
hybrid_search(query, peer_id, limit):

1. Keyword arm (FTS5):
  SELECT o.id, observations_fts.rank
  FROM observations_fts
  JOIN observations o
    ON o.id = observations_fts.rowid
  WHERE observations_fts MATCH ?1
    AND o.is_archived = 0
  ORDER BY observations_fts.rank
  LIMIT 20

2. Semantic arm (sqlite-vec):
  -- embed query via LLM provider
  query_vec = embed(query)

  SELECT rowid, distance
  FROM observation_vec
  WHERE rowid IN (
    SELECT id FROM observations
    WHERE is_archived = 0
  )
  ORDER BY distance
  LIMIT 20

3. RRF merge (in Rust):
  k = 60
  for each result:
    rrf_score = 1/(k + rank_keyword)
              + 1/(k + rank_semantic)
  sort by rrf_score DESC
  return top N
PostgreSQL: tsvector + pgvector + RRF
hybrid_search(query, peer_id, limit):

1. Keyword arm (tsvector):
  SELECT id,
    ts_rank(tsv, plainto_tsquery($1)) AS r
  FROM observations
  WHERE tsv @@ plainto_tsquery($1)
    AND is_archived = FALSE
  ORDER BY r DESC
  LIMIT 20

2. Semantic arm (pgvector):
  -- embed query via LLM provider
  query_vec = embed(query)

  SELECT id,
    embedding <=> $1 AS distance
  FROM observations
  WHERE is_archived = FALSE
  ORDER BY distance
  LIMIT 20

3. RRF merge (in Rust):
  k = 60
  for each result:
    rrf_score = 1/(k + rank_keyword)
              + 1/(k + rank_semantic)
  sort by rrf_score DESC
  return top N

Data Migration: Full Cutover

All agent knowledge state moves to mchact-memory
Data Before (microclaw-storage SQLite) After (mchact-memory, either driver)
Flat memories memories table observations (level=explicit)
Memory vectors sqlite-vec (optional) observation_vec or embedding column
Supersede edges memory_supersede_edges observations.source_ids (DAG)
Reflector runs memory_reflector_runs deriver_runs
Injection logs memory_injection_logs injection_logs
MoA findings subagent_findings (old SQLite) findings (in mchact-memory)
Peers N/A (chat_id only) peers (new)
Peer cards N/A (AGENTS.md) peers.peer_card (new)
File memory AGENTS.md files Deprecated — peer cards replace this

Fallback if memory DB unavailable: No memory at all. Agent works without context — same as a fresh session. No silent degradation, no partial state.
