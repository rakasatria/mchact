How It Works — Runtime Capability Discovery

The Trait Adapts to What's Available
// The unified trait
trait ObservationStore {
    // Core — works on both backends
    fn store_observation(obs: Observation) -> Id;
    fn search(query, strategy: SearchStrategy) -> Vec<Observation>;
    fn get_for_context(chat_id, budget) -> Vec<Observation>;
    fn inject_into_prompt(chat_id) -> String;

    // Capability-gated — graceful degradation
    fn capabilities() -> Capabilities;
    fn get_peer(name) -> Option<Peer>;          // postgres only
    fn get_peer_card(peer_id) -> Option<PeerCard>; // postgres only
    fn trace_reasoning(obs_id) -> Vec<Observation>; // postgres only
    fn dream(peer_id) -> DreamResult;             // postgres only
}

struct Capabilities {
    observation_levels: bool,  // sqlite=false, pg=true
    peer_model: bool,          // sqlite=false, pg=true
    hybrid_search: bool,       // sqlite=maybe, pg=true
    semantic_search: bool,     // sqlite=optional, pg=true
    dreamer: bool,             // sqlite=false, pg=true
    source_dag: bool,          // sqlite=false, pg=true
    contradiction_detect: bool,// sqlite=false, pg=true
}

// Config chooses backend
// storage_backend: "sqlite"    → lightweight, local, existing behavior
// storage_backend: "postgres"  → full observation engine

// The deriver checks capabilities:
if store.capabilities().observation_levels {
    // Extract explicit + deductive observations
} else {
    // Fall back to flat memory reflector
}

Key Insight: Graceful Degradation

Same Code Paths — Different Depth

When running on SQLite:

  • Observations stored as flat memories (category = PROFILE/KNOWLEDGE/EVENT)
  • Reflector extracts facts (existing behavior, unchanged)
  • Search uses FTS5 keyword matching (existing)
  • sqlite-vec for optional semantic search (existing)
  • No peers, no DAG, no dreamer — but everything still works
  • Perfect for: local dev, single-user, lightweight deploys

When running on PostgreSQL + pgvector:

  • Observations stored with full hierarchy (explicit/deductive/inductive/contradiction)
  • Deriver extracts multi-level observations with source attribution
  • Hybrid search: pgvector HNSW + tsvector FTS merged via RRF
  • Peer identity model (observer/observed, user/agent unified)
  • Peer cards (durable biographical summaries)
  • Dreamer runs offline consolidation + pattern discovery
  • Full reasoning DAG with trace_reasoning()
  • Perfect for: production, multi-agent (hermes/clawteam), teams
